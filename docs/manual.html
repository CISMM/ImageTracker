Introduction

ImageTracker is an application designed to track and measure motion in image sequences. Videos often contain motion of several types, and analysis of these motions can be difficult. For example, videos of cells dividing show the cell moving within solution, kinetochores moving back and forth accross the cell equator, and microtubule structures migrating from the equator to the poles. ImageTracker allows for analysis of this type of motion through a multi-scale approach. The video is first stabilized with respect to global motion; flow analysis of smaller scale structures follows.

ImageTracker was written to analyze microscopy videos. Several challenges in analyzing microscopy video means ImageTracker's performance will depend partially on the nature of the video being analyzed. In particular, performance may suffer on videos that have low signal-to-noise ratio, poor feature resolution, sparse temporal sampling (objects move too much between fames), and short object life (features appear and disappear frequently).

Installation

Remove previous versions of ImageTracker from your computer before installing updates. To uninstall ImageTracker, select Start-->All Programs-->NSRG-->ImageTracker-->Uninstall ImageTracker from the start menu.

Note: ImageTracker is a stand-alone application at this time and does not require the NSRG Runtime.

Download and execute the installer from \\nsrg-cs\installers; we recommend that you accept all default parameters in order to conform to NSRG standards. This will create a folder on your harddrive, C:\NSRG\bin\ImageTracker, and will add to your programs menu a folder NSRG\ImageTracker.

Run ImageTracker by selecting Start-->All Programs-->NSRG--> ImageTracker-->ImageTracker.

The main ImageTracker window is shown to the right. The window consists of a menu bar, main canvas, and a status bar. ImageTracker will launch new dialog windows for different purposes, for example, loading files and logging information. Most of these windows may be moved around the screen, risized, or temporarily hidden.

Program information will be written to a Logger window at various points during execution. This information may be useful for making sure the application is still running, or debugging the application if something goes wrong. Click the Hide button on the Logger window to remove it from your desktop. You can recall it by selecting View-->Logger from the ImageTracker menu.

Loading Data

ImageTracker operates on sets of image files. Currently, ImageTracker does not support metamorph stacks or video files. Although ImageTracker technically can read PNG, TIFF, and JPEG image files, the best luck so far has been in reading 8-bit PNG images. Future releases may have support for a wider variety of image formats.

Open a set of image files by selecting the menu item File-->Open. This displays a dialog that allows to manage the set of active image files. Click on the "Add" button to bring up a directory browser. Navigate to the directory that contains your image files, and select the image files you would like to process.

When the ImageTracker active files list is populated, selecting a file in this list will display the image in the main window. You may add or remove files from this list, but all active files must reside in the same directory.

Clicking "OK" on the active files dialog hides the dialog, but does not clear the active files list. You can recall this list at any time by selecting Files-->Open from the menu.

The Open Files dialog controls which image files are active in ImageTracker.

Image Alignment

ImageTracker can stabilize video sequences with respect to certain types of motion through image alignment or registration. In this process an object of interest is identified by the user. ImageTracker will attempt to align all images in the sequence such that this object of interest appears in the same location across all frames. In this context, Alignment, Registration, and Stabilization are all terms that describe the same process.

Launch the registration dialog by selecting Actions-->Register from the ImageTracker menu. The alignment dialog consists of several sections: Thresholding, Smoothing, and Output.

Thresholding and Smoothing alter input images so that alignment is easier to perform. If the preview checkbox on the registration dialog is selected, the first image in the active image list will appear in the ImageTracker main window with the appropriate pre-processing filters applied. Under certain situations, the preview checkbox may be checked but the image in the ImageTracker window is not properly updated. In this case, simply un-check and re-check the preview checkbox. (This may happen if you select an image in the active files dialog and then return to the registration dialog.)

Thresholding

Thresholding selects a range of pixel intensity values to use for registration. In many images, an object of interest will be of a certain intensity, and background objects will be of a different intensity. Selecting an appropriate threshold range will allow the registration process to operate only on the object(s) of interest. At right is an image of a cell that has been properly thresholded to select the mitotic spindle and exclude the cell membrane from the registration.

Thresholding an image controls what intensity ranges are considered during global alignment.

Smoothing

Smoothing an image improves image registration by removing small-scale features. Withouth smoothing, the registration process may get "caught" in local image details, and not find the best global alignment. The goal here is to adjust the degree of smoothing such that the object of interest appears as a cohesive image feature without many small-scale details. An example of an appropriately smoothed and thresholded mitotic spindle is shown to the right.

Smoothing an image selects the scale of the object being aligned.

Output

The output image registration is a list of image transformations that describe how to align images in the sequence. The output section of the registration dialog allows you to select a text file in which to save this information. You may also choose to apply these transformations to the original images and save the registered image sequence. The Transforms and Images checkboxes at the bottom of the registration dialog indicate that you want to save these data.

Select a transforms file by checking the transforms checkbox and clicking on the File button to select an output file. After selecting a transforms file, the directory in which to save registered images is set to the same directory. (Similarly, if you select an images output directory first, a default transforms file is selected for you.)

If you are saving the transfomed image files, the prefix text box indicates the prefix you want to apply to all registered images. This prefix will be prepended to the original image file names. The image at right shows the registration dialog configured to save both transform data and transformed images.

Execution

Once the thresholding, smoothing, and output sections of the registration dialog have been adjusted to your preferences, you are ready to perform the image registration. Clicking on the Next button at the bottom of the dialog will advance the image registration processor by one image pair. That is, ImageTracker will find the optimal transformation between the current and next image in the sequence. No data will be saved until the end of the image sequence is reached, but advancing in this manner for the first one or two images may help you determine if the prefiltering parameters are resonable or give you an indication of how long it will take to align all images in the sequence.

Clicking the Run All button will register all images in the sequence and output the appropriate data at the end of registration. This may take a while...the Logger may provide some information during this process, but sometimes the registration process consumes all available resources, and the information will not be provided.

Flow Computation

ImageTracker provides several methods to determine the image flow from an image sequence. Optic flow is an estimation of real-world motion projected onto the image plane. In the case of 2D motion where the image plane is parallel to the motion plane, optic flow is a good estimate of real-world motion. This scenario is common in microscopy.

To access the optic flow computation methods, select Actions-->Optic Flow from the ImageTracker menu. The options are:

Track Features
Find features of interest within an image and track each of these features across the video sequence; this is good for images that have well-defined features and for motion that may not be spatially smooth
Combined Local Global
Use an iterative algorithm to compute flow estimates at every location in the image; this is good for image sequences containing coherent flow

Each of these options are described in more detail in the following sections.

Flow computation results in a vector image in which each pixel vector represents the computed velocity in the image's x and y directions. Visualization of these images is described in a later section.

Feature-based Tracking

Feature-based tracking attempts to compute motion in images by matching image regions (features) across multiple images. Essential to this technique is the ability to find strong image features—good features to track are image patches that are high in texture content (the local image area has a large gradient in more than one direction). Once image features are detected in an initial image, corresponding features are found in the next image in the sequence through local registration.

The feature tracker dialog allows the user to configure the following parameters:

Max Count
The number of feaures to find; this influences the density of the resulting flow image
Scale
Indicates the size (in pixels) of image features; adjust this to the approximate size of the features of interest in the image
Max Distance
The maximum distance (in pixels) that a feature might move between frames; this defines the image patch search area during the local registration step
Preview
Display a feature image reflecting the selected scale parameter in the ImageTracker image window
Output Images
Specify where to output the vector images representing image flow
Prefix
Specify a prefix to add to input image names to create the output image names

Scale selection is one of the most important parameters in this algorithm. Adjust this paramter such that the feature preview image has well-defined bright patches that correspond to the features of interest. See the images at the right for an example.

Clicking the Run All button will start the feature tracking process. This may take a while. Though some information is logged during this process, the application may become too busy to update the logger window.

CLG Optical Flow

The Combined Local Global (CLG) method of optical flow computation is described in:

Bruhn, Andres, Joachim Weickert, and Christoph Schnorr. Lucas/Kanade meets Horn/Schunck: Combining local and global optic flow methods. International Journal of Computer Vision, 61(3), 211-231, 2005.

The implementation provided in ImageTracker is the single-resolution, simple temporal difference variant. The algorithm is to iteratively estimates optical flow by minimizing an engergy functional that includes a local intensity constancy term (the Lucas/Kanade part) and a global smoothness term (the Horn/Schunck part).

The following parameters on the CLG Optic Flow dialog control the operation of the algorithm:

Spatial Sigma
The spatial smoothing applied to the input images during derivative computation; adjust this to the noise level present in the images
Regularization
Adjusts the constant scaling of the smoothness term in the energy functional; a larger number will result in smoother flow fields, a smaller number will allow more turbulent flow fields
Relaxation
The constant to use during the successive over-relaxation (SOR) step of the algorithm; this affects the speed of the iterative algorithm—Bruhn et al suggest a value in the range 1.90-1.99
Iterations
Specifies the number of iterations to complete when minimizing the error functional; more iterations will presumably provide a lower error residual, but will take longer
Output Images
Specifies where to save the flow field images computed
Prefix
Specifies the prefix to add to input file names to create output file names.

Click the Run All button to start the optic flow computation. Though some information is logged during this process, the application may become too busy to update the logger window.

Visualization

ImageTracker does not provide visualization for computed flow fields (yet). ParaView, a free application from Kitware, provides this functionality. Follow the instructions from Kitware on how to install and run ParaView. The following will help you create a visualization of flow fields with ParaView.

Select File-->Open Data from the ParaView menu. Navigate to and select the flow field file(s) created by ImageTracker (.mha extension). The default visualization for vector image data is a scalar field of vector magnitude intensities. We will create arrow glyphs that will represent the magnitude and direction of the flow field.

Click on the Calculator icon on the ParaView tool bar, or select Filter-->Calculator from the menu. Use the buttons on the calculator parameters tab to create the following expression:

-iHat*ImageFile_0+jHat*ImageFile_1

The ImageFile_x expressions are available under the scalars dropdown box. This expression computes a vector in ParaView from the scalar components stored in the image file. (Note that the negative sign in front of the x-axis expression represents a convention change from that used in ImageTracker.) Once the expression is entered, press the Accept button.

Select Filter-->Glyph from the menu. ParaView will automatically select a vector scale factor appropriate to make the glyphs visible. Press the Accept button.

The default colormap in ParaView is a blue-to-red rainbow color scale. To change this, select the Display tab from the Glyph property control, and then press the Edit Color Map button. Click on the preset colormap button (beneath the lock icon), and select either Greyscale or CIELab Blue to Red. These colormaps may be easier to use to interpret vector data.

It may be helpful to view the original image data along with the vector flow fields. To do this, turn off the visualization of the Calculator filter (the arrow glyphs should remain). Select File-->Open Data from the menu, and select the original image file(s). Note that Paraview does not support TIFF images, so you may have to convert files to a supported format, such as PNG. You may also want to adjust the maximum number of glyphs to display, or their scale factor. The image at the right shows a visualization of mucus flow using the steps thus far.

Sparse Fields

The method described above works well for dense flow fields, such as those created using the CLG Optical Flow method. Sparse fields, such as those created by Feature Tracking, may not contain enough data to effectively represent the image motion without displaying a very large number of vector glyphs. An alternative is to crop the flow field data to restrict the region of interest.

Select the input vector image source from the Selection Window, and select Filter-->Crop from the menu. Adjust the exten to crop using the Parameters tab. Click the Accept button when the crop region corresponds with the desired region of interest.

Select the Calculator filter object, and change its input to ImageClip1. This will change the region over which vector data is calculated and displayed. You may wish to adjust the glyph filter's parameters to increase the number of glyphs displayed or change the scaling factor. The images at the right show the results of cropping the sparse fields generated by feature tracking.
